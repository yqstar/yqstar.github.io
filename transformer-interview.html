🔍 一、Self-Attention 核心机制5 题
1

请用通俗语言解释 Self-Attention（自注意力）的核心思想，并推导其数学公式。

高频必考核心概念
▶
核心回答

Self-Attention 的核心思想是：让序列中的每个 token 都能"看到"所有其他 token，并根据相关性动态加权聚合信息。不再依赖固定窗口或递推结构。

数学推导（Scaled Dot-Product Attention）：

给定输入序列 X ∈ ℝ^(n×d)，通过三个可学习的权重矩阵投影：

Q = X · W_Q    (Query:  我要查什么)
K = X · W_K    (Key:    我有什么标签)
V = X · W_V    (Value:  我包含什么内容)

Attention(Q, K, V) = softmax( Q·K^T / √d_k ) · V

步骤分解：
1. Q·K^T      → 计算每对 token 的"匹配分数" (n×n 注意力矩阵)
2. / √d_k     → 缩放，防止点积过大导致 softmax 梯度消失
3. softmax()  → 将分数归一化为概率分布（每行和为 1）
4. · V        → 按注意力权重对 Value 加权求和

物理直觉：每个 token 发出 Query（"我在找什么"），检查所有 token 的 Key（"我有什么"），匹配度决定从 Value（"我的内容"）中取多少信息。

2

为什么 Self-Attention 中要除以 √d_k（缩放因子）？不除以会怎样？

高频必考细节理解
▶
核心回答

除以 √d_k 是为了防止点积的方差随维度增大而增大，从而避免 softmax 进入梯度极小的饱和区。

  • 不除以 √d_k 的后果：当 d_k 较大时（如 64），Q 和 K 的点积值会很大（方差 ≈ d_k）。大值输入 softmax 后，输出分布变得极度尖锐（接近 one-hot）→ 梯度接近 0 → 模型无法学习。
  • 除以 √d_k 后：点积的方差被归一化到 1 左右，softmax 输出保持适中的熵，梯度能正常反向传播。
# 假设 q, k 是独立的标准正态分布 N(0,1)
# q·k = Σ q_i·k_i,  方差 = d_k
# q·k / √d_k 的方差 = d_k / d_k = 1  ✓  保持稳定

Self-Attention 的复杂度是多少？为什么长序列处理是瓶颈？有哪些优化方向？

高频复杂度分析
▶
核心回答

复杂度：

  • 时间：O(n²·d) — Q·K^T 产生 n×n 矩阵，n 是序列长度
  • 空间：O(n²) — 需要存储注意力矩阵（训练时尤其严重）

优化方向：

方法复杂度思路
Sparse AttentionO(n·k·d)每个 token 只关注局部窗口 + 少数全局 token
Linear AttentionO(n·d²)用核函数替换 softmax，交换计算顺序
FlashAttentionO(n²·d) 但更低的常数IO-aware 的 tiling 算法，减少 HBM 读写
MQA / GQA减少 KV 头数多个 Query 头共享 KV 头，减少 KV cache
Sliding WindowO(n·w·d)限制注意力窗口大小 w（如 Mistral 的 4096）

什么是注意力矩阵（Attention Matrix）的稀疏性假设？它在实践中成立吗？

理论理解
▶
核心回答

注意力矩阵的稀疏性假设认为：每个 token 真正需要关注的 token 只是少数几个，而非全部。注意力矩阵中大部分权重应该接近 0。

  • 实践证明：大多数注意力头学到的模式确实偏向稀疏——有的头关注局部上下文，有的头关注特定语法角色（如动词关注主语），而不是均匀分布。
  • 应用：这是 Sparse Attention（如 Longformer、BigBird）和 Mixture-of-Experts 中稀疏路由的理论基础。
  • 例外：某些层的注意力头确实会展现出较均匀的分布（起到"全局汇总"的作用），因此完全稀疏化会损失部分能力。
🔀 二、Multi-Head Attention5 题
6

为什么需要 Multi-Head Attention（多头注意力）？单头有什么问题？

高频必考核心机制
▶
核心回答

Multi-Head Attention 将 Q、K、V 分别投影到 h 个不同的低维子空间（每个维度 d_k = d_model / h）。核心原因：

  1. 多角度关注：不同头可以关注不同类型的关系——一个头关注语法依存，一个头关注语义相似，一个头关注位置近邻。
  2. 表达能力增强：单头注意力的表达空间受限，多头的并集覆盖了更丰富的特征组合。
  3. 训练稳定性：多头的平均效应可以降低单个注意力头的方差。
MultiHead(Q, K, V) = Concat(head_1, ..., head_h) · W_O

其中 head_i = Attention(Q·W_Qi, K·W_Ki, V·W_Vi)

W_Qi, W_Ki, W_Vi ∈ ℝ^(d_model × d_k)
W_O ∈ ℝ^(h·d_k × d_model)

通常 d_k = d_model / h，如 d_model=512, h=8 → d_k=64

MHA（Multi-Head Attention）、MQA、GQA 三者的区别是什么？各自在什么场景下使用？

高频必考工程优化
▶
核心回答
机制Q 头数KV 头数KV Cache 大小模型质量代表模型
MHAhh最大（h 组 KV）最优GPT-3, BERT
GQAhg (1中等（g 组 KV）接近 MHALlama 2 (70B), Llama 3

  • GQA — 质量和内存的最佳平衡，Llama 2 用 8 个 KV 头配 32 个 Q 头
  • MHA — 质量优先，不在乎推理成本的场景

如何分析不同 Attention Head 学到的模式？它们分别关注什么？

可解释性
▶
核心回答

常见分析角度：

  1. 注意力模式可视化：直接查看注意力权重矩阵，识别每个头关注的 token 对。
  2. 头功能分类（Clark et al., 2019）：
    • 位置头：关注相邻/固定偏移 token
    • 语法头：关注句法依存关系（主语-动词、代词-先行词）
    • 语义头：关注语义相关的 token
    • 分隔头：关注句子边界标记（[SEP]、[CLS]）
  3. 消融实验：剪掉某个头，看下游任务性能下降多少，判断其重要性。
  4. 探针（Probing）：用线性分类器从注意力分布中预测语法/语义特征，验证是否包含该类信息。
💡 研究发现：大量注意力头在推理时是冗余的——剪掉 50% 甚至 80% 的头后模型性能下降很小。这启发了结构化剪枝和动态头选择技术。

Cross-Attention 与 Self-Attention 有什么区别？在 Encoder-Decoder 架构中各起什么作用？

高频
▶
核心回答
维度Self-AttentionCross-Attention
Q 来源当前序列自身Decoder 当前层输出
K, V 来源当前序列自身Encoder 最终输出
作用捕捉序列内部的关系将 Encoder 信息融合到 Decoder
注意力矩阵形状n×n（方阵）m×n（m: Decoder 长度, n: Encoder 长度）

  • Self-Attention 在 Decoder 中：使用 Causal Mask（掩码），每个 token 只能看到自己和之前的 token。
  • Cross-Attention 只在 Decoder 中：让 Decoder 的每个 token 能"查阅"完整的 Encoder 输出（如翻译时，解码器可以关注源语言的任意位置）。

什么是 Casual Attention（因果注意力 / Masked Self-Attention）？为什么 Decoder 必须用它？

基础概念
▶
核心回答

Causal Attention 通过一个上三角遮罩矩阵（Mask），强制第 i 个 token 只能关注第 0 到 i 个 token（当前位置及之前的 token），不能"偷看"未来的 token。

Mask 矩阵 (n=4):        注意力权重 (softmax 后):
[ 0  -∞ -∞ -∞]         [1.0  0    0    0 ]
[ 0   0 -∞ -∞]         [0.3 0.7   0    0 ]
[ 0   0  0 -∞]         [0.2 0.3 0.5   0 ]
[ 0   0  0  0]         [0.1 0.2 0.3 0.4]

为什么 Decoder 必须用它？

  • 自回归生成：Decoder 逐个生成 token，生成时未来的 token 还不存在，自然不能关注。
  • 训练时并行：训练时已知完整序列，但需要用 Mask 模拟推理时的自回归约束，否则就是"作弊"（信息泄露）。
📍 三、位置编码 (Positional Encoding)4 题
11

为什么 Transformer 需要位置编码？Self-Attention 本身为什么不能感知位置？

高频必考核心概念
▶
核心回答

Self-Attention 本质上是置换等变性（Permutation Equivariance）——打乱输入序列的顺序，输出的顺序也相应打乱，但每个位置的输出值不变。

"A 爱 B" 和 "B 爱 A" 在不加位置编码时，Self-Attention 无法区分两者，因为 token 间的注意力权重只取决于内容，不取决于位置。

位置编码的作用：在输入 embedding 上叠加/拼接位置信息，打破置换等变性，让模型能够区分"第一个 token"和"第二个 token"。

12

Sinusoidal Positional Encoding（正弦位置编码）的公式是什么？为什么用 sin/cos？

高频细节推导
▶
核心回答
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

其中 pos = 位置, i = 维度索引, d_model = 模型维度

设计原因：

  1. 不同频率 → 多尺度位置信息：低维度对应低频（波长长，区分大范围位置），高维度对应高频（波长短，区分相邻位置）。
  2. 相对位置的可表达性：由于 sin(a+b) = sin(a)cos(b) + cos(a)sin(b)，PE(pos+k) 可以表示为 PE(pos) 的线性变换，使得模型能学习相对位置关系。
  3. 值域有界：[-1, 1] 范围内，数值稳定。
  4. 可外推：理论上可以推广到训练时未见过的长度，无需重新训练（虽然实际效果有限）。

RoPE（旋转位置编码）的原理是什么？为什么主流大模型（LLaMA、Qwen）都用它？

高频必考现代方法
▶
核心回答

RoPE（Rotary Position Embedding） 通过旋转矩阵将位置信息编码到 Q 和 K 中，使得 Q·K 的点积结果只取决于两个 token 的相对位置。

核心公式：
q_m = R_m · W_Q · x_m     (对位置 m 的 Query 施加旋转)
k_n = R_n · W_K · x_n     (对位置 n 的 Key 施加旋转)

其中 R_m 是一个分块对角旋转矩阵：
R_m = [[cos(mθ₁), -sin(mθ₁)],    # 对第 0,1 维旋转
       [sin(mθ₁),  cos(mθ₁)],
       [cos(mθ₂), -sin(mθ₂)],    # 对第 2,3 维旋转
       ...]

点积结果：
q_m · k_n = (R_m W_Q x_m) · (R_n W_K x_n)
          = x_m^T W_Q^T R_{n-m} W_K x_n  ← 只依赖相对位置 (n-m)

为什么主流模型都用它：

  • 相对位置天然编码：直接内置于 attention 计算中，无需额外位置向量。
  • 外推性好：通过调整 θ 频率（如 NTK-aware scaling），可以较好地扩展到更长的序列。
  • 效率高：实现简单，不增加额外参数，可作为逐元素操作高效计算。

为什么 Transformer 中每个子层都要用残差连接（Residual Connection）+ Layer Normalization？Post-LN 和 Pre-LN 有什么区别？

高频训练稳定性
▶
核心回答

残差连接的作用：

  • 缓解深层网络的梯度消失/爆炸
  • 提供"恒等映射"的快捷路径，让模型更容易学习恒等变换
  • 使深层 Transformer（几十上百层）能够稳定训练

Post-LN vs Pre-LN：

Post-LN (原论文):                Pre-LN (现代做法):
x → SubLayer(x) → Add(x) → LN   x → LN → SubLayer(x) → Add(x)

Output = LN(x + SubLayer(x))    Output = x + SubLayer(LN(x))
维度Post-LNPre-LN
训练稳定性需要 warmup，否则梯度爆炸无需 warmup，天然稳定
收敛速度较快略慢
最终性能可能略好接近或略好

💡 现代大模型几乎全部使用 Pre-LN，因为它消除了对 LR warmup 的依赖，让训练更加简单可靠。

Feed Forward Network (FFN) 在 Transformer 中起什么作用？为什么需要它？

组件理解
▶
核心回答

FFN 是 Transformer 中唯一的位置逐元素（Position-wise）非线性变换。标准形式：

FFN(x) = ReLU(x·W₁ + b₁)·W₂ + b₂     (原始)
FFN(x) = SiLU(x·W_gate) ⊙ (x·W_up) · W_down   (SwiGLU, LLaMA 等使用)

核心作用：

  1. 引入非线性：Self-Attention 本质上是线性的（加权求和），FFN 提供非线性变换能力。
  2. 知识存储：研究表明 FFN 层是 Transformer 存储事实知识的主要位置（类似键值存储器）。W₁ 的每一行可以看作一个"模式检测器"，W₂ 的每一列是响应模式。
  3. 维度变换：先扩展到 4× d_model → 激活 → 压缩回 d_model，提供了更大的表达空间。

Encoder-Only（BERT）、Decoder-Only（GPT）、Encoder-Decoder（T5）三种架构各有什么特点和适用场景？

高频必考架构选型
▶
核心回答
架构注意力方式代表模型最佳场景局限性
Encoder-Only 双向 Self-Attention BERT, RoBERTa, DeBERTa NLU 任务：分类、NER、QA（理解） 不能做文本生成
Decoder-Only 因果（单向）Self-Attention GPT 系列, LLaMA, Claude 文本生成：对话、代码生成、通用任务 某些理解任务不如双向模型
Encoder-Decoder 双向 Enc + 因果 Dec + Cross-Attn T5, BART, 原始 Transformer Seq2Seq：翻译、摘要、输入输出长度不同的任务 参数效率较低（需要两份参数）

为什么 Decoder-Only 主导了 LLM 时代？

  • 统一的"一切皆生成"范式：分类、翻译、QA 都可以格式化为文本生成任务。
  • 训练效率更高：不需要 Encoder，参数集中在生成能力上。
  • Prompt 和 In-Context Learning 天然契合。
  • Scaling 经验最丰富：GPT 系列的 Scaling Law 验证最充分。
⚙️ 五、训练与推理5 题
19

Transformer 的训练过程是怎样的？Teacher Forcing 是什么？

训练机制
▶
核心回答

训练流程：

  1. 输入序列 → Encoder → 上下文表示
  2. 目标序列（右移一位，添加 START token）→ Decoder（使用 Causal Mask）
  3. Decoder 通过 Cross-Attention 融合 Encoder 表示
  4. 输出投影到词汇表 → softmax → 交叉熵损失（与真实目标序列比较）

Teacher Forcing：训练时 Decoder 的输入使用真实的目标序列（ground truth），而不是模型自己上一步预测的 token。

  • 优点：训练稳定、收敛快、可并行计算所有位置。
  • 缺点：训练-推理不一致（Exposure Bias）→ 推理时一步错可能导致后续全错。

缓解 Exposure Bias 的方法：Scheduled Sampling（逐渐用模型自己的输出替换部分 ground truth）。

请详细解释 KV Cache 的原理，以及为什么它能大幅加速自回归推理。

高频必考推理优化
▶
核心回答

KV Cache 是 Decoder 推理时的核心优化：缓存已生成 token 的 Key 和 Value，避免每一步重复计算。

为什么能加速：

  1. 自回归生成时，第 t 步只需要计算新 token 的 Q、K、V
  2. 前 t-1 个 token 的 K 和 V 在上一步已经计算过
  3. 将它们缓存起来，第 t 步只做增量计算：新 Q 与所有历史 K 做 attention
# 无 KV Cache — O(n²·d) 每步
Step 1: 计算 token₀ 的 Q,K,V → attention(Q₀,K₀,V₀)
Step 2: 计算 token₀,₁ 的 Q,K,V → attention(Q₁,[K₀,K₁],[V₀,V₁])  ← 重复计算 K₀,V₀
Step 3: 计算 token₀,₁,₂ 的 Q,K,V → ...                          ← 重复更多

# 有 KV Cache — O(n·d) 每步的新计算
Step 1: 计算 token₀ 的 Q,K,V → attention(Q₀,K₀,V₀) → 缓存 K₀,V₀
Step 2: 计算 token₁ 的 Q,K,V → attention(Q₁,[K₀,K₁],[V₀,V₁]) → 追加缓存 K₁,V₁
Step 3: 计算 token₂ 的 Q,K,V → attention(Q₂,[K₀,K₁,K₂],[V₀,V₁,V₂]) → ...

内存分析：KV Cache 占用 = 2 × n_layers × n_heads × d_head × seq_len × dtype_size。长序列下内存可能非常大（如 LLaMA-7B 在 2048 token 时 KV Cache ≈ 1GB）。

什么是 FlashAttention？它为什么比标准 Attention 更快？

高频前沿优化
▶
核心回答

FlashAttention 是一种 IO-Aware 的精确注意力算法。它不改变数学结果，而是通过优化GPU 内存访问模式来加速。

核心思想 — Tiling（分块计算）：

  1. 标准 Attention 需要将完整的 n×n 注意力矩阵写入 HBM（高带宽显存），这是最大的瓶颈。
  2. FlashAttention 将 Q、K、V 分成小块，在 SRAM（片上共享内存）中逐块计算 softmax，避免将整个注意力矩阵写入 HBM。
  3. 使用 Online Softmax 技巧（tiling + rescaling），保证分块计算与全局计算数学等价。
标准 Attention：
Q,K,V [HBM] → 计算 S=QK^T [写入 HBM] → softmax(S) [读写 HBM]
→ 计算 SV [写入 HBM]     ← 大量 HBM 读写

FlashAttention：
for 每个 Q 块:
    for 每个 K,V 块:
        S_block = Q_block @ K_block^T  [SRAM]
        S_block 做 online softmax       [SRAM]
        累加加权 V_block                 [SRAM]
    输出 O_block [写回 HBM]    ← 只写一次最终结果

效果：训练速度 2-4× 提升，显存 O(n) 而非 O(n²)，允许训练更长的序列。

为什么现在的大模型基本都是 Pre-Norm 而非 Post-Norm？LayerNorm 和 RMSNorm 有什么区别？

训练细节
▶
核心回答

Pre-Norm 主导原因：

  • 训练更稳定，不需要 LR warmup
  • 梯度流更顺畅（norm 在残差分支之前）
  • 对学习率更鲁棒

LayerNorm vs RMSNorm：

LayerNorm(x) = γ · (x - μ) / √(σ² + ε) + β
              需要计算均值和方差，再做缩放和平移

RMSNorm(x)  = γ · x / √(mean(x²) + ε)
              只做缩放，去除均值的中心和偏移

对比：
- RMSNorm 去掉了"去均值"操作，速度提升约 10-15%
- 实验表明去掉去均值不影响性能
- LLaMA、T5、Mistral 等均采用 RMSNorm

如何对 Transformer 做长度外推（Length Extrapolation）？有哪些主流方法？

前沿话题
▶
核心回答

长度外推是指让模型在推理时处理比训练时更长的序列。主要方法：

  1. NTK-aware Scaling（RoPE 专用）：将 RoPE 的频率基数从 10000 放大（如到 500000），改变频率分布，让高频维度更好地适应更长序列。
  2. YaRN（NTK + 温度调节）：在 NTK 基础上对注意力 softmax 加温度系数，缓解长序列的注意力分布过尖问题。
  3. Position Interpolation：将推理时的大位置索引线性缩放到训练时的范围。如训练长度 2048，推理 4096，将位置 4096 映射为 2048。
  4. Sliding Window / 局部注意力：限制有效注意力范围，完全不受长度影响（如 Mistral 的 4096 滑动窗口）。
  5. ALiBi：天然的线性偏置让模型天生支持外推。

SwiGLU 激活函数是什么？为什么现代 LLM（LLaMA、PaLM）用它替代 ReLU？

高频FFN 优化
▶
核心回答

SwiGLU 是一种门控激活函数，结合了 Swish 和 Gated Linear Unit：

ReLU FFN:
FFN(x) = ReLU(x·W₁ + b₁)·W₂ + b₂

SwiGLU FFN:
FFN(x) = (SiLU(x·W_gate) ⊙ (x·W_up)) · W_down
       = Swish(x·W_gate) ⊙ (x·W_up) · W_down

其中 SiLU(x) = x·σ(x)，σ 是 sigmoid
⊙ 表示逐元素乘法（门控）

优势：

  • 门控机制：Swish(x·W_gate) 输出 0~x 之间的门控值，控制信息流。类似 LSTM/GRU 的门控思想。
  • 平滑的非线性：SiLU 处处可导，没有 ReLU 在 x=0 处的不可导问题。
  • 更好的性能：在相同参数量下，SwiGLU 普遍优于 ReLU。

注：使用 SwiGLU 时，通常将 FFN 中间维度缩小（从 4× 调整为约 8/3×），以保持参数量不变。

Mixture of Experts (MoE) 是如何与 Transformer 结合的？解决了什么问题？带来了什么新挑战？

前沿架构
▶
核心回答

MoE 将 Transformer 的 FFN 层替换为多个专家 FFN + 一个路由网络：

标准 FFN:
y = FFN(x)

MoE FFN:
router_logits = x · W_router              # 路由网络计算分数
weights = softmax(TopK(router_logits))    # 选 Top-K 个专家，归一化权重
y = Σ weights_i · Expert_i(x)             # 加权求和选中专家的输出

解决的痛点：增加模型容量而不线性增加计算量。如 Mixtral 8×7B：8 个专家，每次只激活 2 个，总参数 47B，推理成本 ≈ 13B 模型。

新挑战：

  • 负载均衡：路由可能坍缩到少数专家（需要负载均衡损失）
  • 通信开销：专家分布在多卡上时，token 需要在卡间路由
  • 训练稳定性：需要辅助损失（Load Balancing Loss + Router Z-Loss）
  • 显存占用：所有专家都需要加载到显存（虽然只计算部分）

BERT 和 GPT 的 Attention 机制有什么核心区别？为什么 BERT 不能做生成？

Transformer 模型中的 Dropout 通常加在哪些位置？各有什么作用？

正则化
▶
核心回答

Transformer 中 Dropout 的常见位置：

  1. Attention Dropout：在 softmax 后的注意力权重上做 Dropout（随机置零某些注意力连接）。防止过度依赖特定的 token 对。
  2. Residual Dropout：在每个子层的输出加到残差路径之前做 Dropout。防止过拟合，是作用最大的 Dropout 位置。
  3. FFN Dropout：在 FFN 的激活函数之后做 Dropout。
  4. Embedding Dropout：在 Embedding 层之后做 Dropout。

注意：现代大模型（GPT-3+、LLaMA）趋向于减少或不使用 Dropout。因为数据量足够大，加上 Pre-Norm 本身就有正则化作用，Dropout 反而会降低训练效率和最终性能。

请谈谈你对 Transformer 未来发展方向的理解。哪些改进最具潜力？

开放讨论
▶
核心回答

Transformer 仍在快速演进，主要方向：

  1. 状态空间模型（SSM）如 Mamba：将 O(n²) 注意力替换为 O(n) 的状态空间递推，在长序列上具有显著优势。可能成为 Transformer 的补充甚至替代。
  2. 线性注意力：用核函数替换 softmax 来交换计算顺序，实现 O(n) 复杂度。挑战在于质量能否持平 softmax attention。
  3. 稀疏 + MoE：结合稀疏注意力和混合专家，以更低的计算成本达到更大模型的效果。
  4. Infini-Attention：将压缩记忆与因果注意力结合，实现理论上无限的上下文。
  5. 量化 + 高效推理：INT4/FP8 + 更高效的 attention 实现（FlashAttention-3）。
  6. 多模态 Transformer：统一的 Transformer 处理文本+图像+音频+视频。
💡 面试要点：这类问题没有标准答案，重点是展示你对前沿方向的了解广度和思考深度。提到 Mamba/SSM、FlashAttention、MoE 等通常会是加分项。
